# 온누리 드론 지식센터 — 멀티 페이지 웹사이트

모바일 시안을 반응형 멀티 페이지 웹사이트로 구현했습니다.
관리자가 코드 없이 화면에서 콘텐츠를 수정할 수 있습니다.

> **백엔드 연동 추가됨**: 이제 Vercel + Upstash Redis + Resend로
> 콘텐츠 영구 저장과 A/S 접수 이메일 알림을 켤 수 있습니다.
> 백엔드가 없으면 자동으로 localStorage 폴백으로 동작합니다.
> 배포·설정 방법은 **DEPLOY.md** 를 참고하세요.

새로 추가된 파일: `js/api.js`(통신 레이어), `api/` 폴더(서버리스 함수),
`package.json`, `vercel.json`, `.env.example`, `DEPLOY.md`.

## 폴더 구조

```
onnuri/
├─ index.html         홈 (히어로 + 기체선택 + 카테고리)
├─ model.html         기체 상세 (?id=ok20 형태)
├─ manual.html        사용설명서
├─ check.html         비행전 점검 (체크 가능)
├─ fly.html           비행방법
├─ battery.html       배터리 관리
├─ repair.html        정비방법
├─ diagnosis.html     고장진단 (증상별 단계 진단)
├─ education.html     교육영상 (유튜브 임베드)
├─ parts.html         부품도면
├─ as.html            A/S 접수 (폼)
├─ admin.html         ★ 관리자 콘텐츠 편집 페이지
├─ css/style.css      공통 디자인
├─ js/data.js         ★ 모든 콘텐츠 데이터 (한 곳에서 관리)
├─ js/common.js       공통 헤더/푸터/챗봇/QR
└─ images/            실제 사진 넣는 곳
```

## 콘텐츠 수정하는 2가지 방법

### 1) 관리자 페이지 (코드 몰라도 됨) — 추천
1. `admin.html` 접속
2. 비밀번호 입력 (기본: `onnuri`)
3. 탭에서 사이트정보 / 기체 / 카테고리 / 설명서 / 점검 / 증상 / 교육영상 수정
4. 하단 "변경사항 저장" 클릭
5. 다른 페이지를 새로고침하면 즉시 반영

저장 위치는 브라우저(localStorage)입니다. 같은 브라우저에서 본 사람에게만
보이는 데모/원장용 저장이며, 여러 사용자에게 영구 반영하려면 백엔드 연동이 필요합니다.

### 2) 코드 직접 수정 (영구 반영)
`js/data.js` 파일의 값을 고치면 모든 방문자에게 영구 반영됩니다.
실제 운영 배포 시에는 이 방법을 사용하세요.

## 실제 사진으로 교체
1. `images/` 폴더에 사진 파일을 넣습니다.
2. 관리자 페이지(또는 data.js)에서 해당 항목의 이미지 경로에
   `images/파일명.jpg` 를 입력합니다.
3. 경로가 비었거나 파일이 없으면 자동으로 이모지/플레이스홀더가 표시됩니다.

## 관리자 비밀번호 변경
`admin.html` 안의 `const ADMIN_PW = "onnuri";` 값을 원하는 비밀번호로 바꾸세요.
(클라이언트 측 간이 보호이며, 실제 보안이 필요하면 서버 인증이 필요합니다.)

## 실제 운영 전 연동이 필요한 부분
- A/S 접수 폼 → 서버/이메일 전송
- 사용설명서·부품도면 PDF 실제 파일 링크
- 교육영상 유튜브 ID 입력
- 콘텐츠 영구 저장용 백엔드(DB)
- AI 챗봇은 현재 키워드 기반 간이 응답 (실제 AI 연동 가능)

## 로컬에서 보기
폴더에서 `index.html` 을 브라우저로 열면 됩니다.
(파일을 직접 열어도 동작하지만, 일부 브라우저 보안정책상 간단한 로컬 서버 사용을 권장합니다.)
